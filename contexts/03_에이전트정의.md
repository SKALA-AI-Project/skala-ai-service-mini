# 에이전트 정의

## Supervisor

- 역할: 전체 흐름 제어, 품질 점검, 재시도 판단
- 입력: 사용자 목표, 하위 노드 결과
- 출력: 다음 실행 경로, 상태 업데이트

## Web Search Agent

- 역할: 기술·기업별 검색, 소스 수집, 편향 점검
- 입력: 쿼리 세트, 날짜 범위, 다양성 규칙
- 출력: 검색 결과, 풍부도 점수, 편향 점수

## TRL Analysis Node

- 역할: 수집 근거를 TRL 판단으로 변환
- 입력: 검색 결과
- 출력: `trl`, `basis`, `confidence`, `evidence`, `limitation`

## Draft Generation Agent

- 역할: 보고서 7개 섹션 생성, 초안 품질 평가
- 입력: 검색 결과, TRL JSON
- 출력: Markdown 초안, 품질 점수

## HITL Node

- 역할: 검색 다양성 부족 시 수동 검토
- 입력: 편향 검토 실패 결과
- 출력: 진행 승인 또는 재검색 지시

## Formatting Node

- 역할: 승인된 초안을 PDF 보고서로 변환
- 입력: Markdown 초안, TRL 결과
- 출력: 최종 Markdown, PDF 경로, 성공/실패 상태
