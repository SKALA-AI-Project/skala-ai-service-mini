## 1:1 대응표

### 기반 구성 (1단계)

| 설계 요구사항 | 구현 위치 |
|---|---|
| config.py 워크플로우별 설정 분리 | `config.py` — `SearchConfig`, `TrlConfig`, `DraftConfig` |
| perspectives 설정 계층 이동 | `config.SearchConfig.perspectives` |
| 소스 분류 규칙 설정 계층 이동 | `config.SearchConfig.source_type_rules` |
| 품질 임계값 설정 계층 이동 | `config.SearchConfig.bias_score_threshold` / `search_richness_threshold` |
| {목적}_prompt.py 구조 도입 | `prompts/search_prompt.py`, `prompts/trl_prompt.py`, `prompts/draft_prompt.py`, `prompts/quality_prompt.py` |

### 검색 계층 (2단계)

| 설계 요구사항 | 구현 위치 |
|---|---|
| query 생성을 prompts 기반으로 분리 | `prompts/search_prompt.py:build_query()` |
| Tavily 실패 vs 빈 결과 구분 | `agents/web_search_agent.py:_collect_live_results()` — try/except + 빈 결과 로그 분리 |
| 빈 결과 시 mock 대체 금지 | `_collect_live_results()` — `continue`로 처리, mock fallback 없음 |
| 도메인 캡 | `config.SearchConfig.domain_cap` → `_collect_live_results()` |
| source_type 분류 기준 설정 기반 | `_classify_source_type()` — `config.search.source_type_rules` 순회 |
| bias_check / search_richness 정교화 | `_score_results()` — config 임계값 사용 |
| perspectives config 이동 | `config.SearchConfig.perspectives` |

### TRL 분석 (3단계)

| 설계 요구사항 | 구현 위치 |
|---|---|
| 하드코딩 판정 제거, 검색 결과 기반 판정 | `agents/trl_analysis_node.py:_infer_trl(items)` |
| 간접지표 5종 탐지 | `_detect_signals()` × 5 (특허/학회/채용/파트너십/IR) |
| 양산·출하 신호 감지 → TRL 7+ confirmed | `_infer_trl()` 1단계 분기 |
| confidence 5개 지표 기준 산정 | `_calc_confidence(indicator_count, company)` |
| Samsung 공개 보수성 패널티 | `config.TrlConfig.samsung_confidence_penalty` → `_downgrade_confidence()` |
| limitation ±1~2 단계 오차 반영 | `_build_limitation()` |
| TRL 5종 키워드를 config 기반으로 관리 | `config.TrlConfig.*_keywords` |
| TRL 분석 노드에 config 주입 | `TrlAnalysisNode(config)` — report_workflow.py에서 전달 |

## 섹션 이름 변경 이력

design.md의 초안 섹션 이름과 실제 구현 이름이 다르다.
설계 문서를 소급 수정하는 대신 아래 매핑표를 유지한다.

| design.md 명칭 | 구현 명칭 | 비고 |
|---|---|---|
| summary | executive_summary | |
| background | analysis_purpose + analysis_scope | 1개 → 2개로 세분화 |
| tech_status | tech_status | 동일 |
| competitor | investigation_results | 경쟁사 조사 결과 통합 |
| trl_assessment | (investigation_results 내 포함) | |
| insight | conclusion | |
| reference | reference | 동일 |

## 현재 한계

- HITL Node는 골격 구현 (무조건 승인). 실제 담당자 UI 연동 미구현.
- TRL 분석은 검색 결과 텍스트 기반 키워드 매칭. LLM 기반 평가는 `prompts/trl_prompt.py` 참조.
- PDF 생성은 Aspose Words 라이브러리 의존. 환경에 따라 실패 가능.
